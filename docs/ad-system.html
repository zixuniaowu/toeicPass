@toeicpass/ad-system

Backend Frontend Types

A complete, self-contained advertising module providing ad delivery, display, event tracking, and administration. Designed for dependency injection — no framework or HTTP client dependency. Works with any backend storage and any frontend fetch layer.

Key Features

  • 4 ad slot types — banner_top, interstitial, native_feed, reward_video
  • Plan-based targeting — ads target specific subscription plans (free, basic, premium, enterprise)
  • Event tracking — impression, click, dismiss, reward_complete
  • Auto-rotating banner carousel with dot indicators
  • In-feed ad list showing all native_feed ads
  • Fullscreen interstitial with countdown auto-close
  • Rewarded video with completion callback
  • Admin CRUD panel with statistics dashboard
  • Google AdSense compatibility via slot mapping
  • 10 built-in seed ads including 3 Hugging Face promotions

IAdStore Interface

interface IAdStore {
  adPlacements: AdPlacement[];
  adEvents: AdEvent[];
  persistSnapshot(): void;
}

The host application must implement this interface. Backing store can be in-memory arrays, PostgreSQL, MongoDB, Redis, or any database.

AdApiFunctions Interface

interface AdApiFunctions {
  fetchAds(slot?: string): Promise<AdPlacement[]>;
  recordAdEvent(placementId: string, eventType: string): Promise<void>;
}

All user-facing components (AdBanner, NativeFeedAd, InterstitialAd, RewardVideoAd) accept an api: AdApiFunctions prop.

AdminAdApiFunctions Interface

interface AdminAdApiFunctions extends AdApiFunctions {
  fetchAdminAds(): Promise<AdPlacement[]>;
  fetchAdStats(): Promise<AdStats>;
  createAd(input: CreateAdInput): Promise<AdPlacement>;
  updateAd(id: string, input: UpdateAdInput): Promise<AdPlacement>;
  deleteAd(id: string): Promise<void>;
}

AdManagerView accepts api: AdminAdApiFunctions for the full admin experience.

Google AdSense Integration

The GoogleAdUnit component renders AdSense ads when the NEXT_PUBLIC_ADSENSE_CLIENT environment variable is set. The ad-provider.ts file maps ad slots to AdSense slot IDs:

// web/lib/ad-provider.ts
const ADSENSE_SLOTS: Record<AdSlot, string> = {
  banner_top: "1234567890",
  native_feed: "2345678901",
  interstitial: "3456789012",
  reward_video: "4567890123",
};

Dependency Injection Pattern

Backend: Store Injection

The AdService constructor accepts an IAdStore implementation and an AdServiceConfig object. No database driver or ORM is bundled.

const service = new AdService(store, {
  generateId: () => uuid(),
  nowIso: () => new Date().toISOString(),
});

Frontend: API Function Injection

Components receive an api prop containing async functions. They never call fetch, axios, or any HTTP client directly.

<AdBanner
  locale="en"
  showAds={true}
  api={{
    fetchAds: myFetchFn,
    recordAdEvent: myTrackFn,
  }}
/>

Backend Usage

import { AdService, DEFAULT_AD_SEEDS } from "@toeicpass/ad-system";
import type { IAdStore } from "@toeicpass/ad-system";

// 1. Implement IAdStore
const store: IAdStore = {
  adPlacements: [],
  adEvents: [],
  persistSnapshot() { fs.writeFileSync("ads.json", JSON.stringify(this.adPlacements)); },
};

// 2. Create AdService
const adService = new AdService(store, {
  generateId: () => crypto.randomUUID(),
  nowIso: () => new Date().toISOString(),
});

// 3. Seed (runs only when empty)
adService.seedIfEmpty(DEFAULT_AD_SEEDS);

// 4. Fetch ads for a user on the "free" plan
const bannerAds = adService.getAdsForUser("free", "banner_top");
const allAds = adService.getAdsForUser("free");

// 5. Track events
adService.recordAdEvent(bannerAds[0].id, "user_abc", "impression");
adService.recordAdEvent(bannerAds[0].id, "user_abc", "click");

// 6. Admin
const stats = adService.getAdStats();
const newAd = adService.createAd({
  slot: "native_feed",
  title: "New Product Launch",
  linkUrl: "https://example.com",
  ctaText: "Learn More",
  targetPlans: ["free", "basic"],
});
adService.updateAd(newAd.id, { priority: 95 });
adService.deleteAd(newAd.id);

Frontend Usage

import { AdBanner, NativeFeedAd, InterstitialAd, RewardVideoAd, AdManagerView }
  from "@toeicpass/ad-system/web";
import type { AdApiFunctions, AdminAdApiFunctions } from "@toeicpass/ad-system/web";

// Define API functions
const adApi: AdApiFunctions = {
  fetchAds: (slot) =>
    fetch(`/api/v1/ads?slot=${slot}`, { headers: { Authorization: `Bearer ${token}` } })
      .then(r => r.json()),
  recordAdEvent: (id, type) =>
    fetch("/api/v1/ads/event", {
      method: "POST",
      headers: { "Content-Type": "application/json", Authorization: `Bearer ${token}` },
      body: JSON.stringify({ placementId: id, eventType: type }),
    }).then(() => {}),
};

// Banner carousel at top of page
<AdBanner locale="en" showAds={true} slot="banner_top" api={adApi} />

// In-feed ads within content
<NativeFeedAd locale="en" showAds={true} api={adApi} />

// Fullscreen interstitial (triggered on navigation, etc.)
<InterstitialAd locale="en" showAds={true} api={adApi} autoCloseMs={8000} />

// Rewarded video
<RewardVideoAd
  locale="en"
  showAds={true}
  api={adApi}
  onRewardEarned={() => grantBonus()}
  onSkip={() => showMessage("Reward skipped")}
/>

// Admin panel (requires admin API functions)
<AdManagerView locale="en" api={adminApi} />

Thin Wrapper Pattern

In the toeicPass project, each frontend component is wrapped in a ~27-line file that binds authentication tokens to the API functions:

// apps/web/components/ads/AdBanner.tsx
"use client";
import { useMemo } from "react";
import * as api from "../../lib/api";
import { AdBanner as AdBannerBase } from "@toeicpass/ad-system/web";

export function AdBanner({ locale, token, tenantCode, slot, showAds }) {
  const adApi = useMemo(() => {
    const opts = { token, tenantCode };
    return {
      fetchAds: (s) => api.fetchAds(opts, s),
      recordAdEvent: (id, type) => api.recordAdEvent(id, type, opts),
    };
  }, [token, tenantCode]);

  return <AdBannerBase locale={locale} showAds={showAds} slot={slot} api={adApi} />;
}

This keeps the package decoupled from any specific auth mechanism while the app wrapper handles token injection.

Cross-Project Reuse

  1. Copy packages/ad-system/ into your new project's packages directory
  2. Add it to your package.json workspaces and run npm install
  3. Backend: Implement IAdStore, create an AdService instance, expose API endpoints
  4. Frontend: Import components, create an AdApiFunctions object pointing to your API
  5. Configure TypeScript paths and test framework aliases
Zero framework lock-in. The backend service is a plain TypeScript class. Frontend components are React-based but communicate with the host via injected functions, making the API layer fully pluggable.
