@toeicpass/conversation-ai

Backend Frontend Types Voice

An AI-powered voice conversation module for English language practice. Provides a conversation engine with built-in TOEIC scenarios, Google Gemini API integration with automatic rule-based fallback, and a complete full-screen voice chat UI with speech recognition and synthesis.

Key Features

  • 8 built-in TOEIC scenarios covering office, restaurant, airport, hotel, shopping, meeting, phone, and interview contexts
  • Google Gemini 2.0 Flash integration for natural AI responses
  • Automatic fallback to rule-based engine when no API key is configured
  • Grammar corrections included in AI responses
  • Conversation suggestions to help learners continue
  • Full-screen voice chat UI optimized for mobile
  • Speech-to-text via Web Speech API (SpeechRecognition)
  • Text-to-speech via Web Speech API (SpeechSynthesis)
  • No external dependencies — plain TypeScript backend, React frontend

ConversationService

The core backend class providing scenario listing and AI reply generation.

MethodParametersReturnsDescription

generateReply()ConversationReplyInputPromise<ConversationReplyResult>Generates an AI response using Gemini API or rule-based fallback. Includes grammar corrections and conversation suggestions.

AI Engine Selection

Google Gemini (Primary)

When geminiApiKey is provided, the service calls the Gemini 2.0 Flash API with a specialized English tutoring prompt. Responses include natural conversation, grammar corrections, and follow-up suggestions.

Rule-Based Fallback

When no API key is available, the service uses a built-in rule engine that generates contextually appropriate responses based on the scenario's context field. No network calls required.

ConversationView Component

A full-screen React component providing the complete voice chat experience.

Features

  • Full-screen chat interface with large, readable text
  • Press-and-hold microphone button for voice input
  • Automatic text-to-speech for AI responses
  • Inline grammar correction highlights

ConversationApiFunctions Interface

interface ConversationApiFunctions {
  fetchScenarios(): Promise<ConversationScenario[]>;
  sendReply(payload: ConversationReplyInput): Promise<ConversationReplyResult>;
}

Voice Features

Browser Native APIs — No third-party libraries required. Voice features use the Web Speech API, which is supported in Chrome, Edge, Safari, and most modern browsers.

Speech-to-Text (Input)

Uses the SpeechRecognition API. The user presses and holds the microphone button to speak. Recognition results are displayed in the chat as user messages. The speech-recognition.d.ts file provides TypeScript type definitions for the browser API.

Text-to-Speech (Output)

Uses the SpeechSynthesis API. AI responses are automatically read aloud after being received. The component selects an English voice from the available system voices.

Dependency Injection Pattern

Backend: Configuration Injection

The ConversationService constructor accepts an optional ConversationServiceConfig. The Gemini API key and model name are configurable. No network library is bundled.

const service = new ConversationService({
  geminiApiKey: "your-key",
  modelName: "gemini-2.0-flash",
});

Frontend: API Function Injection

The ConversationView component receives an api prop with two async functions. It never calls fetch or any HTTP client directly.

<ConversationView
  locale="en"
  api={{
    fetchScenarios: myFetchFn,
    sendReply: myReplyFn,
  }}
/>

Backend Usage

import { ConversationService } from "@toeicpass/conversation-ai";

// Create service — Gemini is used when API key is provided
const service = new ConversationService({
  geminiApiKey: process.env.GEMINI_API_KEY, // optional
  modelName: "gemini-2.0-flash",            // optional, this is the default
});

// List all available scenarios
const scenarios = service.listScenarios();
// → [{ id: "office-small-talk", title: "Office Small Talk", ... }, ...]

// Generate an AI reply
const result = await service.generateReply({
  scenarioId: "restaurant-order",
  text: "I'd like to order the grilled salmon, please.",
  history: [
    { role: "assistant", content: "Welcome! What can I get for you today?" },
  ],
});

console.log(result.content);
// → "Great choice! The grilled salmon is one of our most popular dishes. ..."

console.log(result.corrections);
// → [] (no errors in this input)

console.log(result.suggestions);
// → ["Could I also see the wine menu?", "What sides come with that?"]

Without API Key (Rule-Based Fallback)

// No API key → automatic rule-based fallback
const service = new ConversationService({});
// OR
const service = new ConversationService();

const result = await service.generateReply({
  scenarioId: "office-small-talk",
  text: "How was your weekend?",
  history: [],
});
// Still returns content, corrections, and suggestions
// using built-in rules based on scenario context

Frontend Usage

import { ConversationView } from "@toeicpass/conversation-ai/web";
import type { ConversationApiFunctions } from "@toeicpass/conversation-ai/web";

// Define API functions pointing to your backend endpoints
const api: ConversationApiFunctions = {
  fetchScenarios: () =>
    fetch("/api/v1/conversation/scenarios", {
      headers: { Authorization: `Bearer ${token}` },
    }).then(r => r.json()),

  sendReply: (payload) =>
    fetch("/api/v1/conversation/reply", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Authorization: `Bearer ${token}`,
      },
      body: JSON.stringify(payload),
    }).then(r => r.json()),
};

// Render the full-screen voice chat
function ConversationPage() {
  return <ConversationView locale="en" api={api} />;
}

Thin Wrapper Pattern

In the toeicPass project, the component is wrapped in a ~26-line file that binds authentication tokens:

// apps/web/components/conversation/ConversationView.tsx
"use client";
import { useMemo } from "react";
import * as api from "../../lib/api";
import { ConversationView as ConversationViewBase } from "@toeicpass/conversation-ai/web";
import type { ConversationApiFunctions } from "@toeicpass/conversation-ai/web";

export function ConversationView({ locale, token, tenantCode }) {
  const conversationApi = useMemo<ConversationApiFunctions>(() => ({
    fetchScenarios: () => api.fetchConversationScenarios({ token, tenantCode }),
    sendReply: (payload) => api.fetchConversationReply(payload, { token, tenantCode }),
  }), [token, tenantCode]);

  return <ConversationViewBase locale={locale} api={conversationApi} />;
}

Cross-Project Reuse

  1. Copy packages/conversation-ai/ into your new project's packages directory
  2. Add it to package.json workspaces and run npm install
  3. Backend: Create a ConversationService instance, expose /scenarios and /reply endpoints
  4. Frontend: Import ConversationView, create a ConversationApiFunctions object pointing to your API
  5. Configure TypeScript paths and test framework aliases
Minimal integration effort. The backend service is a single class with no dependencies. The frontend is one React component with one CSS module. Total integration typically requires implementing just 2 API endpoints and one wrapper component.
